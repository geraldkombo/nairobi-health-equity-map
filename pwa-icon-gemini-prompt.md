# PWA Icon Design — Gemini Prompt

I need a production-quality PWA icon set for the **Kenya Health Equity Map** (https://geraldkombo.github.io/kenya-health-equity-map/). The icon should work as a home-screen app icon, favicon, and social tile.

## Constraints

1. **Zero external tools** — I can only write SVG files by hand and generate PNG from them. No Figma, no Illustrator, no image editor, no Sharp/Canvas in Node.
2. **SVG inline or file-based** — Icons must be pure SVG or generated by a minimal script.
3. **Small file footprint** — PWA icons should be under 2 KB.
4. **PNG fallback** — I need a Node script that converts SVG to PNG at 192x192 and 512x512. Use sharp if needed (installable), or a pure-JS approach.
5. **Maskable** — The icon must look good within the Android maskable safe zone (central ~80% of canvas).

## Current icons (problems)

- `icon-192.svg`: Kenya flag emoji on cream bg — emoji renders differently per OS, often pixelated or missing
- `icon-512.svg`: Brown gradient bg + white plus/cross shape + "HEALTH" text — decent but not polished
- `icon-192.png` & `icon-512.png`: Generated by a raw-PNG-spec script. Blocky 3-color horizontal stripes (black/red/green) — looks terrible

## Design requirements

- **Symbol**: Something that represents health equity / maps / Kenya. Candidates:
  - Stylized map pin with a cross/plus inside
  - Kenya map outline (too detailed for small sizes?)
  - Cross/caduceus with a rising arrow
  - Abstract "E" for Equity combined with a heartbeat or map pin
- **Colors**: Use the project's exact brand palette: `#78350F` (primary brown), `#EA580C` (accent orange), `#FDFBF7` (cream bg), `#292524` (text dark)
- **Target**: Professional, public-health, credible — not cartoonish
- **Favicon version**: A simplified 16x16 version that is recognizable at small sizes (ideally just the symbol, no text)

## Deliverables

1. The text of the SVG for each size:
   - `icon-192.svg` (192x192, favicon + small tile)
   - `icon-512.svg` (512x512, large tile + splash)
   - `icon.svg` (scalable, used as fallback)
2. The favicon markup for `layout.tsx` (inline SVG in `link rel="icon"`)
3. For the PNG generation, give me the updated `generate-icons.mjs` script that uses **sharp** (or another approach) to render the SVG at the required sizes with proper anti-aliasing
4. Any `manifest.json` changes needed for maskable icon purposes

## Important

- The SVG must be self-contained (no external fonts, no external images)
- Keep the design simple enough to render well at 192x192 and recognizable at 48x48
- Test your design mentally: would it look good on an Android home screen, iOS home screen, and browser tab?

Please output the complete code for each file and the favicon markup.
